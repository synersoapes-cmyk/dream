## 1. 项目背景 & 目标

### 1.1 项目背景

梦幻西游作为运营超过20年的国民级MMORPG，拥有极其复杂的数值体系。玩家在装备选择、属性加点、伤害优化等方面面临大量计算需求，且游戏内缺乏直观的装备对比和伤害预估工具。

### 1.2 用户痛点

|   |   |
|---|---|
|痛点|描述|
|**装备对比难**|更换一件装备需要手动计算所有面板变化，无法直观看到收益|
|**属性加点纠结**|不同加点方案的面板差异难以量化，尤其跨种族比较|
|**伤害估算靠经验**|打怪/PK伤害只能靠"感觉"，无法精确预估|
|**装备价值难评估**|高价装备是否物有所值，缺乏数据支撑|
|**信息录入繁琐**|手动抄录装备属性容易出错且耗时|

### 1.3 产品目标

|   |   |
|---|---|
|阶段|目标|
|**短期（V1-V3）**|实现属性模拟 + 装备管理 + 伤害计算的核心功能闭环|
|**中期（V4-V5）**|智能推荐装备、自动最优加点方案|
|**长期（V6+）**|社区数据共享、AI辅助决策、藏宝阁比价|

### 1.4 成功指标

|   |   |   |
|---|---|---|
|指标|目标值|衡量方式|
|属性计算准确率|≥95% 与游戏内面板一致|与实际游戏截图对比验证|
|OCR识别成功率|≥90% 装备属性正确提取|用户确认反馈|
|单次操作耗时|≤3秒完成一件装备录入|从上传到识别完成|
|用户留存|周回访率 ≥40%|本地存储活跃数据|

## 2. 用户与使用场景

### 2.1 目标用户画像

|   |   |   |   |
|---|---|---|---|

|**数值党**|129-175级|热衷研究公式，精打细算每一点属性|精确属性计算、多方案对比|
|**日常PVE玩家**|129-175级|关注性价比，需要判断装备是否值得买|装备价值评估、简易面板查看|

### 2.2 核心使用场景

**场景1：装备购买决策**

> 我在藏宝阁看到一把武器¥15万，想知道换上后面板变化多少、伤害提升多少、值不值这个价。

**场景2：属性加点优化**

> 我的角色175级龙宫，想对比"全魔加点"和"1体3魔1敏"两种方案的面板差异。

**场景3：PK伤害预估**

> 武神坛比赛前，想知道我的法术对一个9000血、1200法防的对手能打多少伤害。

**场景4：装备套装规划**

> 我目前用混沌兽套，想看看如果换成碧波套或者菩提套，整体属性有什么变化。

### 2.3 用户使用路径

```Plaintext
首次使用：
  上传人物面板截图 → OCR识别 → 确认属性 → 逐件上传装备截图 → OCR识别 → 确认
  → 系统自动计算面板属性 → 保存为方案

日常使用：
  打开工具 → 选择已有方案 → 替换某件装备 → 查看面板变化 → 决策

高级使用：
  创建多个方案 → 横向对比 → 伤害模拟 → 导出报告
```

## 6. 数据与算法设计（核心）

### 6.1 属性转换公式（五维 → 面板）

> 数据来源：[梦幻西游官方属性页](https://xyq.163.com/introduce/newcomer/xs007.html)、[3DM属性换算公式2025](https://ol.3dmgame.com/gl/289033.html)、社区验证

#### 6.1.1 三族转换系数对照表

|   |   |   |   |   |
|---|---|---|---|---|
|属性|面板属性|人族系数|仙族系数|魔族系数|
|体质 → 气血|HP|**5**|**4.5**|**6**|
|体质 → 速度|SPD|0.1|0.1|0.1|
|体质 → 灵力|MDMG|0.3|0.3|0.3|
|魔力 → 魔法|MP|**3**|**3.5**|**2.5**|
|魔力 → 灵力|MDMG|**0.7**|**0.7**|**0.7**|
|力量 → 伤害|DMG|**0.66**|**0.56**|**0.76**|
|力量 → 速度|SPD|0.1|0.1|0.1|
|力量 → 灵力|MDMG|0.4|0.4|0.4|
|耐力 → 防御|DEF|**1.5**|**1.6**|**1.4**|
|耐力 → 速度|SPD|0.1|0.1|0.1|
|耐力 → 灵力|MDMG|0.2|0.2|0.2|
|敏捷 → 速度|SPD|**0.7**|**0.7**|**0.7**|
|敏捷 → 躲避|-|1|1|1|

> **关键差异**：魔族体质→气血最高(6)，仙族魔力→魔法最高(3.5)、耐力→防御最高(1.6)，魔族力量→伤害最高(0.76)。

#### 6.1.2 面板属性计算公式

**基础面板公式**（以人族为例）：

```Plaintext
HP（气血）= (体质 × 5 + 100) × (1 + 强身术等级/100) + 装备气血加成
MP（魔法）= (魔力 × 3 + 80) × (1 + 冥想等级/100) + 装备魔法加成
DMG（物伤）= 力量 × 0.66 + 装备伤害加成
MDMG（法伤/灵力）= 魔力 × 0.7 + 体质 × 0.3 + 力量 × 0.4 + 耐力 × 0.2 + 装备灵力加成
DEF（防御）= 耐力 × 1.5 × (1 + 强壮等级/100) + 装备防御加成
SPD（速度）= (敏捷 × 0.7 + 体质 × 0.1 + 力量 × 0.1 + 耐力 × 0.1) × (1 + 神速等级%) + 装备速度加成
```

**通用公式模板**：

```Plaintext
最终面板 = (基础属性点 × 种族系数 + 基础常数) × (1 + 被动技能加成%) + 装备直接加成
```

其中：

- `基础属性点` = 人物加点 + 装备附加五维 + 潜能果 + 粽子
    
- `种族系数` = 见 6.1.1 表格
    
- `基础常数` = HP为100，MP为80，其他为0
    
- `被动技能加成` = 技能等级 / 100（即每级+1%）
    
- `装备直接加成` = base_stats中的对应数值之和
    

#### 6.1.3 被动技能加成公式

|   |   |   |   |   |
|---|---|---|---|---|
|被动技能|影响属性|每级加成|上限|公式|
|**强身术**|HP 气血|+1%|140级|`HP × (1 + 强身术等级/100)`|
|**冥想**|MP 魔法|+1%|140级|`MP × (1 + 冥想等级/100)`|
|**强壮**|DEF 防御|+1%|140级|`DEF × (1 + 强壮等级/100)`|
|**神速**|SPD 速度|固定+1.5点/级|40级|`SPD + 神速等级 × 1.5`（最高+60速度）|

> **强身术示例**：89级强身术 → 气血增加89%。若基础气血1000，则最终气血 = 1000 × 1.89 = 1890。

> **神速说明**：与其他被动技能不同，神速是固定值加成（每级+1.5速度），而非百分比。最高40级，满级加60点速度。

> ⚠️ **需要你确认**：神速的加成方式在不同来源中说法不一（有说固定+1.5/级，有说百分比加成）。你游戏内的神速是哪种？

#### 6.1.4 人物修炼加成

|   |   |   |
|---|---|---|
|修炼类型|每级效果|最高等级|
|攻击修炼|攻击时HP伤害结果 +2% +5点|25级（飞升后）|
|防御修炼|受到物理攻击HP伤害结果 -2% -5点|25级（飞升后）|
|法术修炼|法术HP伤害结果 +2% +5点，封系成功率+2%|25级（飞升后）|
|抗法修炼|受到法术伤害结果 -2% -5点，被封几率-2%|25级（飞升后）|

> 修炼条件：人物等级 ≥ 修炼等级 × 5 + 20

### 6.2 伤害计算公式

#### 6.2.1 物理伤害公式

```Plaintext
基础攻防差 = 面板伤害 - 敌方防御
  → 若 攻防差 < 面板伤害 × 10%，则取 面板伤害 × 10%

技能伤害 = 攻防差 × 技能倍率

修炼加成 = 技能伤害 × (1 + 攻击修炼等级 × 2%) + 攻击修炼等级 × 5
修炼减免 = 修炼加成 × (1 - 敌方防御修炼等级 × 2%) - 敌方防御修炼等级 × 5

最终伤害 = max(1, 修炼减免) × 随机系数(0.95~1.05)

暴击伤害 = 最终伤害 × 1.5（基础暴击倍率）
```

#### 6.2.2 法术伤害公式

```Plaintext
法术基础伤害 = 面板法伤(灵力) × 技能系数 + 武器伤害/4

灵力差 = 己方灵力 - 敌方灵力 × 0.9

法术伤害 = 法术基础伤害 + 灵力差 + 30

修炼加成 = 法术伤害 × (1 + 法术修炼等级 × 2%) + 法术修炼等级 × 5

最终伤害 = max(1, 修炼加成) × 随机系数(0.95~1.05)
```

#### 6.2.3 暴击机制

```Plaintext
物理暴击率 = min(95%, 物理暴击等级 / 1750 × 100%)
法术暴击率 = min(95%, 法术暴击等级 / 1750 × 100%)
暴击倍率 = 1.5 + 暴击等级 × 0.001

期望伤害 = 普通伤害 × (1 - 暴击率) + 暴击伤害 × 暴击率
```

#### 6.2.4 简化计算模型（当前版本实现）

考虑到完整公式需要大量参数（修炼等级、五行克制、阵型等），当前V3版本使用简化模型：

```JavaScript
// 物理技能
base = 面板DMG × skill.multiplier
reduced = max(1, base - 敌方DEF × (1 - skill.penetration))
critRate = min(0.95, PCrit / 1750)
critMult = 1.5 + PCrit × 0.001
avgDmg = reduced × (1 - critRate) + reduced × critMult × critRate

// 法术技能
base = 面板MDMG × skill.multiplier
reduced = max(1, base - 敌方MDEF × (1 - skill.penetration))
// 暴击逻辑同上
```

- **专注**：法术攻击时的魔法消耗降低为0的几率提高。
    
- **永不磨损**：装备永远不消耗耐久度。
    
- **珍宝/伪装/迷踪/坚固**：出售给商人价格提高/改变外形等辅助特效。

## 7. OCR & 数据录入方案

### 7.1 技术方案

|   |   |
|---|---|
|项目|说明|
|引擎|**Claude Vision API**（claude-sonnet-4-20250514）|
|调用方式|前端直接调用 Anthropic REST API|
|请求头|`x-api-key` + `anthropic-dangerous-direct-browser-access: true`|
|图片格式|JPG/JPEG/PNG/GIF/WEBP/BMP → base64编码|
|返回格式|纯 JSON（结构化装备数据）|

### 7.2 OCR识别流程

```Plaintext
用户上传截图（支持1-13张批量）
    │
    ▼
显示 OCR Loading Overlay（进度条 + 队列）
    │
    ▼
逐张发送到 Claude Vision API
    │
    ├── 图片类型 = 人物面板
    │     → 返回 {type:"character_panel", data:{race, baseStats, passiveSkills, ...}}
    │     → 自动填充左侧属性面板
    │
    └── 图片类型 = 装备截图
          → 返回 {type:"equipment", slot:"head", data:{name, level, base_stats, ...}}
          → 自动填入对应装备槽位
    │
    ▼
用户确认/手动修正 → 实时重算面板属性
```

### 7.3 图片要求规范

|   |   |
|---|---|
|要求|说明|
|来源|藏宝阁截图 或 游戏内装备详情截图|
|分辨率|≥720px 宽度（保证文字清晰）|
|内容|需包含完整装备信息区域|
|格式|JPG/PNG 均可，≤5MB|

### 7.4 异常处理

|   |   |
|---|---|
|异常|处理方式|
|OCR返回不完整|提示用户手动补充缺失字段|
|槽位识别错误|允许用户手动选择正确槽位|
|API调用失败|显示错误信息，支持重试|
|图片模糊/遮挡|提示用户重新截图|
|识别到多件装备|取主要装备，忽略背景信息|
